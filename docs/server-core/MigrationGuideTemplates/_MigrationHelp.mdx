## SDK Migration Checklist

1. Update Dependencies

- Remove the old SDK package/module from your project.
- Add the new SDK package/module.
- Include any required platform-specific dependencies for your environment.

2. Update Imports / Requires

- Update import or require statements to reference the new SDK namespace or module.
- Remove any references to the old SDK's namespace or structure.

3. Update Initialization

- Global instance signatures have changed or not yet supported. Update the signature or replace global initialization with instance-based initialization.
- If needed, use the appropriate builder or configuration pattern for setting options and creating user objects.

4. Update User Creation

- Migrate to the new format for creating user objects.
- If needed, use the builder pattern or updated constructor to set user properties.

5. Update Method Calls

- Replace static/global method calls with instance methods on the new SDK object, or update the global instance signature (if supported).
- Update any suffixes or prefixes from method names.
- Update method signatures and usage according to the new API.

6. Test Thoroughly

- Verify that feature gates, experiments, and dynamic configs return expected values.
- Ensure that event logging and exposure tracking are functioning correctly.
- Confirm proper SDK initialization and error handling.
