# Autocapture on the Web

The JavaScript, React, and Angular SDKs support autocapture, which automatically captures events and their attributes without requiring you to manually instrument your code.

## Attributes Captured

### Click Events

| Property | Description |
| --- | --- |
| `click_target_tag` | The HTML tag of the clicked element |
| `click_target_id` | The ID attribute of the clicked element |
| `click_target_class` | The class attribute of the clicked element |
| `click_target_text` | The text content of the clicked element |
| `click_target_href` | The href attribute if the clicked element is a link |
| `click_x` | The x-coordinate of the click |
| `click_y` | The y-coordinate of the click |

### Page View Events

| Property | Description |
| --- | --- |
| `page_url` | The current page URL |
| `page_title` | The page title |
| `referrer` | The referrer URL |
| `utm_source` | UTM source parameter |
| `utm_medium` | UTM medium parameter |
| `utm_campaign` | UTM campaign parameter |
| `utm_term` | UTM term parameter |
| `utm_content` | UTM content parameter |

### Form Submission Events

| Property | Description |
| --- | --- |
| `form_id` | The ID of the submitted form |
| `form_class` | The class of the submitted form |
| `form_action` | The action URL of the form |
| `form_method` | The HTTP method of the form (GET/POST) |

### Performance Events

| Property | Description |
| --- | --- |
| `load_time` | Page load time in milliseconds |
| `dom_interactive_time` | Time until DOM is interactive |
| `first_contentful_paint_time` | Time to first contentful paint |
| `largest_contentful_paint_time` | Time to largest contentful paint |
| `cumulative_layout_shift` | Cumulative layout shift score |

## Event Filtering

You can filter autocapture events programmatically using the `eventFilterFunc` option in the `StatsigAutoCapturePlugin`:

<GitHubEmbed url="https://raw.githubusercontent.com/statsig-io/js-client-monorepo/main/samples/react/src/samples/precomputed-client/sample-precomp-auto-capture-filter.tsx" />

The `eventFilterFunc` receives the `AutoCaptureEvent` object and should return `true` to keep the event or `false` to filter it out. You can filter based on `eventName`, `value`, or any property in the `metadata`.

## Common Metadata Captured

Statsig automatically captures comprehensive metadata with each autocapture event to provide rich context for analysis:

#### Page and Environment Metadata:

| Property | Description | Example Value |
| --- | --- | --- |
| `metadata.title` | Page title from `<title>` tag | "Homepage - My Website" |
| `metadata.current_url` | Full current URL | "https://example.com/page?utm_source=google" |
| `metadata.hostname` | Domain name of the current page | "example.com" |
| `metadata.pathname` | Path portion of the URL | "/products/shoes" |
| `metadata.user_agent` | Browser user agent string (truncated to 200 chars) | "Mozilla/5.0 (Windows NT 10.0; Win64; x64)..." |
| `metadata.locale` | Browser language setting | "en-US" |
| `metadata.timezone` | User's timezone | "America/New_York" |
| `metadata.timezone_offset` | Timezone offset in minutes | -300 |
| `metadata.timestamp` | Event timestamp in milliseconds | 1692123456789 |

#### Screen and Viewport Data:

| Property | Description | Example Value |
| --- | --- | --- |
| `metadata.screen_width` | Screen width in pixels | 1920 |
| `metadata.screen_height` | Screen height in pixels | 1080 |
| `metadata.viewport_width` | Browser viewport width in pixels | 1200 |
| `metadata.viewport_height` | Browser viewport height in pixels | 800 |

#### Network Information:

| Property | Description | Example Value |
| --- | --- | --- |
| `metadata.effective_connection_type` | Connection speed estimate | "4g" |
| `metadata.rtt_ms` | Round-trip time in milliseconds | 50 |
| `metadata.downlink_mbps` | Download speed in Mbps | 10.5 |
| `metadata.save_data` | Whether user has data saver enabled | true |

#### Referrer and Traffic Source Data:

| Property | Description | Example Value |
| --- | --- | --- |
| `metadata.referrer` | Full referrer URL | "https://google.com/search?q=example" |
| `metadata.referrer_domain` | Domain of the referrer | "google.com" |
| `metadata.referrer_path` | Path of the referrer URL | "/search" |
| `metadata.searchEngine` | Detected search engine | "google" |
| `metadata.searchQuery` | Search query from referrer | "example search term" |
