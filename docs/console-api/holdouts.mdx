# console/v1/holdouts

<Rapidoc id="segments" entity="segments"/>

## Authorization

All requests must include the **STATSIG-API-KEY** field in the header. The value should be a **Console API Key** which can be created in the Project Settings on [console.statsig.com/api_keys](https://console.statsig.com/api_keys)

## Responses

All responses will contain a **message** field and a **data** field.

- **message** is a simple string explaining the result of the operation.
- **data** will either be a [Holdout](#holdout-object) object or an array of [Holdout](#holdout-object) objects, depending on the request you made.

## Create a Holdout

```bash
curl --request POST 'https://api.statsig.com/console/v1/holdouts' \
--header 'STASIG-API-KEY: console-xxXxxXXXXxxxxXxx' \
--header 'Content-Type: application/json' \
--data-raw '{
    "name": "a holdout",
    "description": "helpful summary of what this holdout does",
    "idType": "userID"
}'
```

**Reponse JSON** : A single [Holdout](#holdout-object) object

```json
{
  "message": "Holdout created successfully.",
  "data": {
    "id": "a_holdout",
    "isEnabled": true,
    "description": "helpful summary of what this holdout does",
    ...
  }
}
```

## Reading All Holdouts

```bash
curl --request GET 'https://api.statsig.com/console/v1/holdouts' \
--header 'STATSIG-API-KEY: console-xxXxxXXXXxxxxXxx'
```

**Reponse JSON** : An array of [Holdout](#holdout-object) objects

```json
{
  "message": "Holdouts listed successfully.",
  "data": [
    {
      "id": "a_holdout",
      "isEnabled": true,
      "description": "helpful summary of what this holdout does"
      ...
    },
    {
      "id": "b_holdout",
      "isEnabled": true,
      "description": "similar to a_holdout but it has a b instead"
      ...
    }
  ]
}
```

## Reading a Single Holdout

```bash
curl --request GET 'https://api.statsig.com/console/v1/holdouts/a_holdout' \
--header 'STATSIG-API-KEY: console-xxXxxXXXXxxxxXxx'
```

**Reponse JSON** : A single [Holdout](#holdout-object) object

```json
{
  "message": "Holdout read successfully.",
  "data": {
    "id": "a_holdout",
    "isEnabled": true,
    "description": "helpful summary of what this holdout does",
    ...
  }
}
```

## Updating a Holdout

```bash
curl --request POST 'https://api.statsig.com/console/v1/holdouts/a_holdout' \
--header 'STATSIG-API-KEY: console-xxXxxXXXXxxxxXxx' \
--header 'Content-Type: application/json' \
--data-raw '{
  "isEnabled": false,
  "description": "an updated summary of what this holdout does",
  "passPercentage": 5,
  "gateIDs": ["a_gate"],
  "experimentIDs": ["a_experiment"],
  "layerIDs": ["a_layer"],
  "isGlobal": true
}'
```

**Reponse JSON** : A single [Holdout](#holdout-object) object

```json
{
  "message": "Holdout updated successfully.",
  "data": {
    "id": "a_holdout",
    "isEnabled": false,
    "description": "an updated summary of what this holdout does",
    ...
  }
}
```

## Deleting a Holdout

```bash
curl --request DELETE 'https://api.statsig.com/console/v1/holdouts/a_holdout' \
--header 'STATSIG-API-KEY: console-xxXxxXXXXxxxxXxx'
```

**Reponse JSON** : Only a message is returned for this request

```json
{
  "message": "Holdout deleted successfully."
}
```

## Objects

<a name="holdout-object"></a>

### Holdout

- **id** | The name that was originally given to the holdout on creation but formatted as an ID ("A Holdout" -> "a_holdout").
- **isEnabled** | Is the holdout enabled.
- **description** | A brief summary of what the holdout is being used for.
- **idType** | The type of ID which the holdout is based on.
- **isGlobal** | Is the holdout being applied to all new gates.
- **lastModifierName** | The Statsig Username of the last modifier of this holdout.
- **lastModifierID** | The Statsig UserID of the last modifier of this holdout.
- **passPercentage** | The percentage of custumers, on the range of 0%-10%, who won't see the new features.
- **gateIDs** | An array of gatesIDs which this holdout is to be applied.
- **experimentIDs** | An array of experimentsIDs which this holdout is to be applied.
- **layerIDs** | An array of layersIDs which this holdout is to be applied.

```json
{
  "id": "my_holdout",
  "isEnabled": true,
  "description": "a summary of what this holdout does",
  "idType": "userID",
  "isGlobal": true,
  "lastModifierName": "CONSOLE API",
  "lastModifierID": "kcj8gnkvk2kcj9d",
  "passPercentage": 2,
  "gateIDs": ["a_gate", "b_gate"],
  "experimentIDs": ["my_experiment"],
  "layerIDs": ["my_layer"]
}
```
