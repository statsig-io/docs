# console/v1/gates/{gate_id}/overrides

## Authorization

All requests must include the **STATSIG-API-KEY** field in the header. The value should be a **Console API Key** which can be created in the Project Settings on [console.statsig.com/api_keys](https://console.statsig.com/api_keys)

## Responses

All responses will contain a **message** field and a **data** field.

- **message** is a simple string explaining the result of the operation.
- **data** will be a [Gate Overrides](#gate-overrides-object) object.

## Read Overrides for a Gate

```bash
curl --request GET 'https://api.statsig.com/console/v1/gates/a_gate/overrides' \
--header 'STATSIG-API-KEY: console-xxXxxXXXXxxxxXxx'
```

**Reponse JSON** : A [Gate Overrides](#gate-overrides-object) object

```json
{
  "message": "Gate Overrides read successfully.",
  "data": {
    "passingUserIDs": [],
    "failingUserIDs": []
  }
}
```

## Update Overrides for a Gate

```bash
curl --request POST 'https://api.statsig.com/console/v1/gates/a_gate/overrides' \
  --header 'STATSIG-API-KEY: console-xxXxxXXXXxxxxXxx' \
  --header 'Content-Type: application/json' \
  --data-raw '{
    "passingUserIDs": [
        "passing-user"
    ],
    "failingUserIDs": [
        "failing-user"
    ]
}'
```

**Reponse JSON** : A [Gate Overrides](#gate-overrides-object) object

```json
{
  "message": "Gate Overrides read successfully.",
  "data": {
    "passingUserIDs": ["passing-user"],
    "failingUserIDs": ["failing-user"]
  }
}
```

## Objects

<a name="gate-overrides-object"></a>

### Gate Overrides

- **passingUserIDs** | An array of UserIDs that will be forced to pass the gate.
- **failingUserIDs** | An array of UserIDs that will be forced to fail the gate.

```json
{
  "passingUserIDs": ["passing-user"],
  "failingUserIDs": ["failing-user"]
}
```
