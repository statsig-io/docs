Statsig Server SDKs evaluate gate, config, experiment checks locally, on your server.

In general, Server SDKs follow the same pattern:

1. Initialize the SDK.  The SDK will reach out to Statsig for the most up to date definition of all of your feature gates, dynamic configs, experiments, autotunes, and layers
2. Evaluate `checkGate`/ `getConfig` / `getExperiment` calls locally
3. Batch exposure logs with calls to `logEvent` and periodically flush logs to Statsig servers
