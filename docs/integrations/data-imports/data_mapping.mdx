## Overview

Statsig requires certain data schema in order for proper processing. We support 3 different types of datasets to be ingested into our platform:

1. Custom Events
2. Precomputed Metrics
3. Exposure Events

---

### Custom Events

Events that are emitted by your application to measure the ongoing impact of your features and experiments.

#### Required

| Column         | Description                            | Rules                                                                                                                    |
| -------------- | -------------------------------------- | ------------------------------------------------------------------------------------------------------------------------ |
| time_stamp     | The unix time your event was logged at | Not null                                                                                      |
| event_name     | The name of the event                  | Not null. Length < 128 characters                                                                                        |
| unit_id        | Unique unit identifier                 | e.g., User ID, Stable ID, etc. Can have multiple. |

#### Optional

| Column         | Description                            | Rules                                                                                                                    |
| -------------- | -------------------------------------- | ------------------------------------------------------------------------------------------------------------------------ |
| timeuuid       | A unique timeuuid for the event        | This should be a timeuuid, but using a unique id will suffice. If not provided, the table defaults to generating a UUID. |
| user           | A user json object.                    | See user object below                                                                                                                |
| event_value    | The value of the event                 | Length < 128 characters                                                                                                  |
| event_metadata | Metadata about the event               | Not null. Length < 16384 characters. Json-formatted - leave empty if none                                                |
| event_version  | The version of this event              |                             

The user object is a stringified JSON representation. An example might look like:

```json
{
  "os": "Mac OS X",
  "os_version": "10.15.7",
  "browser_name": "Electron",
  "browser_version": "11.5.0",
  "ip": "1.1.1.1",
  "country": "KR",
  "locale": "en-US",
  "userID": "aaaaa-aaaaa-aaaaa-aaaaa",
  "custom": {
    "locale": "en-US",
    "clientVersion": "23.10.23.10",
    "desktopVersion": "11.5.0"
  },
  "customIDs": {
    "deviceId": "aaaaa-aaaaa-aaaaa-aaaaa",
    "stableID": "aaaaa-aaaaa-aaaaa-aaaaa"
  }
}
```

### Precomputed Metrics

Precomputed metrics to augment your feature and experiment analysis in Statsig.

#### Required

| Column       | Description                                                                                                                   | Rules                                                                                                                    |
| ------------ | ----------------------------------------------------------------------------------------------------------------------------- | ------------------------------------------------------------------------------------------------------------------------ |
| unit_id      | The unique user identifier this metric is for. This might not necessarily be a user_id - it could be a custom_id of some kind |                                                                                                                          |
| id_type      | The id_type the unit_id represents.                                                                                           | Must be a valid id_type. The default Statsig types are user_id/stable_id, but you may have generated custom id_types     |
| date         | Date of the daily metric                                                                                                      |                                                                                                                          |
| metric_name  | The name of the metric                                                                                                        | Not null. Length < 128 characters                                                                                        |
| metric_value | A numeric value for the metric                                                                                                | Metric value <br/><b>OR</b><br/> Both of numerator/denominator need to be provided for Statsig to process the metric.             |
| numerator    | Numerator for metric calculation                                                                                              | If present along with a denominator in any record, the metric will be treated as ratio and only calculated for users with non-null denominators. |
| denominator  | Denominator for metric calculation                                                                                            | If present along with a numerator in any record, the metric will be treated as ratio and only calculated for users with non-null denominators.                                                                                           |

#### Optional

| Column       | Description                                                                                                                   | Rules                                                                                                                    |
| ------------ | ----------------------------------------------------------------------------------------------------------------------------- | ------------------------------------------------------------------------------------------------------------------------ |
| timeuuid     | A unique timeuuid for the event                                                                                               | This should be a timeuuid, but using a unique id will suffice. If not provided, the table defaults to generating a UUID. |
