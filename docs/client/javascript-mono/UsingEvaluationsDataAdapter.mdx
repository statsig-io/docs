The `EvaluationsDataAdapter` is the default `StatsigDataAdapter` used by the `PrecomputedEvaluationsClient`.
It handles caching and fetching values from Statsig. If required, you can create your own custom `StatsigDataAdapter`
and provide it to the Statsig client via [StatsigOptions.dataAdapter](/client/javascript-sdk#statsig-options).

## Overview

### Synchronous Behavior (Cache)

When callling `PrecomputedEvaluationsClient.initalizeSync`, the `EvaluationsDataAdapter` will load values
from Cache and provide them to the client. The client will also call refresh in the background via
`EvaluationsDataAdapter.getDataAsync`, leading to values lagging until
the next time `initializeSync` or `updateUserSync` is called.

### Asynchronous Behavior (Network)

When callling `PrecomputedEvaluationsClient.initializeAsync`, the `EvaluationsDataAdapter` will load values
from Cache and provide them to the client. The client will then block on a call to `EvaluationsDataAdapter.getDataAsync`,
allowing the ability to await the latest values from Statsig.

### Bootstrapping

Bootstrapping allows you to provide data for a given StatsigUser. This can be useful if you are are running a
Statsig Server SDK on your backend and do not wish to make unneccesary network calls (See [statsig-node](/server/nodejsServerSDK#bootstrap)'s getClientInitializeResponse).
This approach can also be useful if you are bundling values with your application and want to load from a local file.
