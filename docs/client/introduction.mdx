Statsig Client SDKs evaluate gate, config, experiment checks up front for a particular user, and then serve all checks synchronously from this evaluation.

In general, Client SDKs follow the same pattern:

1. Initialize the SDK.  The SDK will reach out to Statsig and evaluate all feature gates, dynamic configs, experiments, autotunes, and layers for the given user
2. Evaluate `checkGate`/ `getConfig` / `getExperiment` calls from the values fetched in (1)
3. Batch exposure logs with calls to `logEvent` and periodically flush logs to Statsig servers
