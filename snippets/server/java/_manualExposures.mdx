## Manual Exposures

Manual exposure logging gives you control over when exposure events are sent to Statsig. This is useful for preventing logging exposures on feature checks that don't result in the user actually experiencing the feature.

### Gates

Check a gate without automatically logging an exposure:

```kotlin
val passed_or_failed = Statsig.checkGateWithExposureLoggingDisabled(user, "a_gate")
```

Manually log the gate exposure:

```kotlin
Statsig.manuallyLogGateExposure(user, "a_gate")
```

### Dynamic Configs

Get a config without automatically logging an exposure:

```kotlin
val config = Statsig.getConfigWithExposureLoggingDisabled(user, "awesome_product_details")
```

Manually log the config exposure:

```kotlin
Statsig.manuallyLogConfigExposure(user, "awesome_product_details")
```

### Experiments

Get an experiment without automatically logging an exposure:

```kotlin
val titleExperiment = Statsig.getExperimentWithExposureLoggingDisabled(user, "new_user_promo_title")
```

Manually log the experiment exposure:

```kotlin
Statsig.manuallyLogConfigExposure(user, "new_user_promo_title")
```

### Layers

Get a layer without automatically logging an exposure:

```kotlin
val layer = Statsig.getLayerWithExposureLoggingDisabled(user, "user_promo_experiments")
val promoTitle = layer.getString("title", "Welcome to Statsig!")
```

Manually log the layer parameter exposure:

```kotlin
Statsig.manuallyLogLayerParameterExposure(user, "user_promo_experiments")
```
