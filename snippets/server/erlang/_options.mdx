You can specify optional parameters in the [application config](https://www.erlang.org/doc/design_principles/applications.html#configuring-an-application).

<ResponseField name="statsig_api_key" type="string" required>
  A server secret api key from the statsig console.
</ResponseField>

<ResponseField name="statsig_api" type="string" default="https://statsigapi.net/v1/">
  The endpoint to use for network requests to load gates/experiments and flush log events.
</ResponseField>

<ResponseField name="http_client" type="module" default="hackney_client">
  A network client for the SDK to issue requests. Must use the `http_client` behavior.
</ResponseField>

<ResponseField name="statsig_polling_interval" type="integer" default="60000">
  How frequently to poll for changes to gates/experiments in your statsig project (in milliseconds).
</ResponseField>

<ResponseField name="statsig_flush_interval" type="integer" default="60000">
  How frequently to flush event and exposure logs for your statsig project (in milliseconds).
</ResponseField>

<ResponseField name="statsig_flush_batch_size" type="integer" default="500">
  Maximum batch size of events.
</ResponseField>

<ResponseField name="statsig_environment_tier" type="string" default="undefined">
  The environment tier to set for this instance of the statsig erlang sdk. Options: "production", "staging", "development" or your own custom tiers.
</ResponseField>
