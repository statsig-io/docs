## Expo Specific Setup

### Create a Client

Creating a Statsig client when using Expo has a slight twist when compared to pure JavaScript environments.

Because storage in Expo is async (See [AsyncStorage](https://docs.expo.dev/versions/latest/sdk/async-storage/)), a warming step is required to load any cache values from disk. This warming performs some `AsyncStorage.getItem` operations that block the `StatsigProviderExpo` from rendering.

```typescript
import { StatsigClient } from '@statsig/js-client';
import { StatsigProviderExpo } from '@statsig/expo-bindings';
import { warmCachingFromAsyncStorage } from '@statsig/react-bindings';
import AsyncStorage from '@react-native-async-storage/async-storage';

// Create the client
const myStatsigClient = new StatsigClient(
  'client-sdk-key',
  { userID: 'a-user' },
  {
    networkConfig: {
      preventAllNetworkTraffic: false,
    },
  }
);

// Start the warming process
const warming = warmCachingFromAsyncStorage(
  myStatsigClient,
  AsyncStorage
);
```

<Tip>
View the full example on [GitHub](https://github.com/statsig-io/js-client-monorepo/blob/main/samples/expo/src/app/snippets/snippet-expo-warming.tsx)
</Tip>

### Render the StatsigProvider

Once the Statsig client is created and the warming step started, we can then pass them into the `StatsigProviderExpo` component for rendering.

```typescript
export function App(): JSX.Element {
  return (
    <StatsigProviderExpo client={myStatsigClient} cacheWarming={warming}>
      {/* Render your App */}
    </StatsigProviderExpo>
  );
}
```
