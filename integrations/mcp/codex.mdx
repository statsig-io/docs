## Overview

You can set up the Statsig MCP server with OAuth on Codex CLI and Codex IDE Extension today.

## Installation
Navigate to `~/.codex/config.toml` and add the below: 

```toml
[mcp_servers.statsig]
url = "https://api.statsig.com/v1/mcp"
trust_level = "trusted"

[features]
rmcp_client = true
```

## Authentication via OAuth

You'll need to sign in to your Statsig account and authorize the MCP server to access your Statsig project by running the below:
```bash
codex mcp login statsig
```

Once configured, you should see the Statsig MCP server listed as active in the Codex extension's MCP servers panel.

<Accordion title="Using API Key instead of OAuth">
While we recommend setting up Statsig MCP via OAuth, you can also authenticate with your Console API key. Navigate to `~/.codex/config.toml` and add the below:

```toml
[mcp_servers.statsig]
command = "npx"
args = ["--yes", "mcp-remote", "https://api.statsig.com/v1/mcp", "--header", "statsig-api-key: console-YOUR-CONSOLE-API-KEY"]
trust_level = "trusted"
```

Replace `console-YOUR-CONSOLE-API-KEY` with your actual Statsig Console API key, which you can retrieve [here](https://console.statsig.com/api_keys). Ensure your API key has the right permissions â€” read-only keys can view data, while write keys can make changes to your project!

</Accordion>

## Using Statsig MCP with Codex

With Statsig MCP configured in any Codex environment, you can:

- **Explore Experiments**: "List all my active experiments"
- **Manage Gates**: "What gates are currently stale?"
- **Configure Dynamic Configs**: "Show me the configuration for the dynamic config 'dynamic-config'"
- **Get Insights**: "Show me details about the experiment called 'new-checkout-flow'"
